# 분실신고·부정거래 환수 SOP

> **AUDIENCE**: ADMIN  **CATEGORY**: SECURITY_OPS  **BASIS**: 전자금융거래법 §9, 전기통신금융사기 피해방지 특별법

## 1. 분실신고 접수 (카드·통장·인감·OTP)

### 1-1. 접수 채널·시간
| 채널 | 접수 가능 시간 | 즉시 차단 조치 |
|---|---|---|
| 콜센터 | 24시간 | 5분 이내 |
| 모바일 앱 | 24시간 | 즉시 (자동) |
| 영업점 창구 | 영업시간 (09:00~16:00) | 10분 이내 |
| ATM 분실보고 | 24시간 | 30분 이내 |

### 1-2. 접수 시 확인 항목
1. 고객 본인 확인 (주민번호 끝 4자리 + 비밀번호)
2. 분실 일시·장소·경위
3. 분실 후 거래 발생 여부
4. 부정사용 의심 거래 즉시 보고 (있으면 §2 절차)

### 1-3. 처리 결과
- 카드·통장: 즉시 사용정지 → 재발급 (창구 1영업일, 우편 5영업일)
- OTP: 시리얼 차단 → 신규 발급
- 인감: 신고일 즉시 효력 발생 → 새 인감 등록

## 2. 부정거래 환수 절차 (전자금융거래법 §9)

### 2-1. 1단계 — 부정거래 신고
1. 고객 신고 또는 FDS 시스템 자동 감지
2. 「부정거래확인서」 작성 — 거래 내역·고객 진술
3. 즉시 계좌 거래제한 (`LIMITED_ACCOUNT_YN='Y'`)

### 2-2. 2단계 — 1차 조사 (5영업일)
- 거래 IP·디바이스·시간대 분석
- 인증 수단 사용 흔적 (OTP·생체·간편비밀번호) 확인
- 고객 본인 인증 미사용 거래 vs 사용 거래 분류

### 2-3. 3단계 — 책임 분담 결정

| 사유 | 책임 |
|---|---|
| 은행 시스템 결함·해킹 | **은행 100% 환급** |
| 고객 비밀번호·OTP 등 인증정보 유출 (제3자 도용) | **은행 100% 환급** (전자금융거래법 §9 ①) |
| 고객 중대 과실 (예: 비밀번호 공유) | 은행·고객 안분 (보통 50:50) |
| 고객 고의 (자작극 포함) | 환급 거부 + 경찰 신고 |

### 2-4. 4단계 — 환급
- 책임 분담 결정 후 **20영업일 이내** 환급
- 미결정 시 가환급 가능 (사후 조사 결과로 정산)

### 2-5. 5단계 — 외부 보고
- 1억 원 이상 사고 → 금감원 즉시 보고
- 분기별 사고 통계 → 한국은행·금융위 보고

## 3. 사고 사례별 대응

### 3-1. SMS 본인인증 가로채기 (스미싱)
- 인증번호 자동입력 앱 의심 → 단말 포렌식 검사 요구
- 통신사 USIM 변경 이력 확인

### 3-2. 보이스피싱 후 자발적 송금
- 전기통신금융사기 피해방지법 §10 채권 소멸 절차 안내
- 수취 은행에 지급정지 신청

### 3-3. 명의도용 신규 가입
- 신분증 위조 여부 진위확인 API 재호출
- 즉시 계좌 해지 + KYC 기록 보존 5년

### 3-4. 내부직원 부정거래
- 즉시 직원 시스템 접근 차단
- 감사부 단독 조사 — 영업점 분리
- 형사고소 + 임직원 징계

## 4. 환수금 회수
- 부정거래자에 대한 구상 청구 (민법 §750)
- 형사 고소 후 추징 절차
- 자체 회수 불가 시 손해 정리 → 손익계산서 반영

## 5. 시스템 자동 감지
- **AI_FDS_DECISION** 룰 발동 + ML 점수 80+ 자동 보류
- 사용자에게 NOTIFICATION 발송 — "의심 거래 발생, 확인 요청"
- 24시간 내 본인 확인 없으면 자동 보류 해제 또는 연장 (정책)
