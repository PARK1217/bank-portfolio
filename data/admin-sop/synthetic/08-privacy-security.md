# 개인정보보호·보안 SOP

> **AUDIENCE**: ADMIN  **CATEGORY**: PRIVACY  **BASIS**: 개인정보보호법, 신용정보법, 전자금융거래법

## 1. 개인정보 수집 원칙

### 1-1. 최소 수집 원칙
- 거래 목적 달성에 필요한 최소 정보만
- 별도 동의 시에만 마케팅 정보·외부 제공

### 1-2. 동의 항목 (필수 vs 선택 명확 구분)
| 구분 | 항목 | 기간 |
|---|---|---|
| 필수 | 신원확인 정보 (이름·주민번호·연락처) | 거래 종료 후 5년 |
| 필수 | 거래 내역 | 10년 |
| 선택 | 마케팅 정보 | 동의 철회 시까지 |
| 선택 | 신용정보 외부 조회 | 거래 종료 후 3년 |

### 1-3. 수집 출처 명시
- 본인 직접 입력
- 신용정보원·NICE·KCB 등 신용평가사
- 행정안전부 진위확인 API
- 통신사 본인인증

## 2. 정보 처리 권한 (3등급)

### 2-1. Level 1 (일반 직원)
- 본인 담당 고객의 거래 조회만
- 다른 직원 담당 고객 — 「조회 사유서」 작성 후 가능

### 2-2. Level 2 (지점장·본부 부서)
- 지점·부서 단위 통합 조회
- 다지점 조회 — 사후 감사

### 2-3. Level 3 (CISO·감사부)
- 전사 조회 권한
- 시스템 로그·접근 이력 조회
- 「특별조사명령」 근거 보존

### 2-4. 모든 접근 — `ADMIN_AUDIT_LOG` 자동 적재
- EMPLOYEE_NO + ACTION_CD + TARGET_TABLE + TARGET_ID 필수
- 5년 보존 (개인정보보호법)

## 3. 개인정보 유출 사고 대응

### 3-1. 발생 인지 (시간 단위 SOP)

#### 0~1시간
1. 정보보호최고책임자(CISO) 즉시 보고
2. 유출 범위 1차 파악 — 건수·항목·발생경로
3. 시스템 격리 (접근 차단)

#### 1~24시간
4. 본부 비상대책반 가동
5. 한국인터넷진흥원(KISA) 사고 신고 (개인정보보호법 §34)
6. 정확한 유출 범위·원인 분석

#### 24~72시간
7. 정보주체에게 통지 — 우편·이메일·SMS
   - 통지 내용: 유출 항목·일시·원인·은행 조치사항
8. 금감원·금융위 보고

#### 72시간 이후
9. 재발 방지 대책 수립·시행
10. 손해배상 검토 — 1인당 손해액 산정

### 3-2. 통지 의무 (개인정보보호법 §34)
- 1,000명 이상 유출 시 — 즉시 통지 + 한국인터넷진흥원 신고
- 100명~999명 — 영업일 5일 이내 통지
- 100명 미만 — 영업일 10일 이내 통지

## 4. 보안 정책

### 4-1. 패스워드 정책
- 영문 대소문자 + 숫자 + 특수문자 4종 중 3종 이상
- 8자 이상 16자 이하
- 90일 주기 변경 강제
- 5회 실패 시 잠금 (`LOGIN_FAIL_COUNT` 5 이상)

### 4-2. 다중인증 (MFA)
- 1차: 비밀번호 또는 SIMPLE_PIN
- 2차: OTP (TOTP·pyotp 검증) — 6자리, 30초 변경
- 3차 (1억 원 이상 거래): 생체인증 또는 본인확인 SMS

### 4-3. 세션·토큰
- JWT 만료 — 30분
- silent refresh — 활동 감지 시 자동 갱신 (mousemove·keydown 등)
- 로그아웃 — JWT jti 블랙리스트 즉시 적재 (`is_jti_revoked`)

## 5. 위반 시 제재

### 5-1. 임직원
- 정보 외부 유출 — 형사고소 + 즉시 해고
- 권한 외 조회 — 징계 (감봉~정직)
- 시스템 미사용 (개인 USB 등) — 시말서 + 교육

### 5-2. 회사
- 최대 매출 3% 과징금 (개인정보보호법 §39의15)
- CEO 형사 책임 — 5년 이하 징역 또는 5천만 원 이하 벌금

## 6. 시스템 연동
- 모든 admin 조회 → `ADMIN_AUDIT_LOG` 자동 적재
- 권한별 접근 — `EMPLOYEE_MASTER.AUTH_LEVEL_CD` (ADMIN/AUDIT/GENERAL)
- 본인 외 고객 조회 — `DELEGATION` 테이블에서 위임권한 검증
- 1억 원 이상 거래 — `ACCOUNT_LIMIT_CHANGE_REQUEST` 7일 cooling-off 적용
