# 결제망 장애 대응 SOP (금융결제원·한국은행 거액결제망)

> **AUDIENCE**: ADMIN  **CATEGORY**: SYSTEM_OPS  **BASIS**: 한국은행 결제시스템 운영규정, 금융결제원 결제시스템 규약

## 1. 결제망 구조
| 결제망 | 운영기관 | 대상 거래 | 운영시간 |
|---|---|---|---|
| **금융결제원 소액결제망** | 금융결제원 | 1억 원 미만 타행 이체 | 24시간 (점검 02:00~04:00 예외) |
| **한국은행 거액결제망 (한국은행 거액결제망)** | 한국은행 | 1억 원 이상 타행 이체 | 평일 09:00~17:30 |
| **자동이체 결제망 (자동이체)** | 금융결제원 | 자동이체·계좌이체 | 영업일 09:00~17:00 |
| **CD/ATM 공동망** | 금융결제원 | 타행 ATM 입출금 | 24시간 |
| **체크카드/신용카드** | VAN사·BC카드 | 카드 결제 | 24시간 |

## 2. 장애 단계별 정의

### Level 1 — 부분 지연 (응답 시간 평소 2배 이상)
- 영향: 일부 거래 지연
- 사용자 알림 X
- 운영팀 모니터링 강화

### Level 2 — 부분 장애 (실패율 5% 이상)
- 영향: 일부 거래 실패
- 사용자 알림: 화면 상단 배너
- 본부 IT 운영실 즉시 보고

### Level 3 — 전체 장애 (응답 없음 또는 실패율 50% 이상)
- 영향: 거래 전면 중단
- 사용자 알림: 푸시·SMS·홈 화면 팝업
- 본부 비상대책반 즉시 가동

### Level 4 — 장기 장애 (3시간 이상 미복구)
- 영향: 영업 중단
- 외부 보고: 금감원·한국은행·금융위 즉시 신고
- 대체 채널 안내 (수기 처리·다른 은행 송금 등)

## 3. 장애 인지 시 1차 대응 (10분 이내)

1. 모니터링 시스템 알림 수신 (`외부 연동 상태` 테이블 DEGRADED/DOWN 자동 감지)
2. 영업점·콜센터 안내 — 시스템 점검 화면 노출
3. 본부 IT 운영실 → 결제망 운영기관(금융결제원·BOK) 직통 연락
4. 「장애 발생 보고서」 1차 작성

## 4. 거래 처리 임시 조치

### 4-1. 금융결제원 소액결제망 장애
- 진행 중인 PENDING 거래 → 자동 재시도 (최대 3회·간격 5분)
- 신규 거래 — 사용자에게 "잠시 후 재시도" 안내
- 30분 이상 미복구 → 거래 보류 + 사용자 환불 절차 안내

### 4-2. 한국은행 거액결제망 장애 (거액결제망)
- 운영시간 외 거래 시도 → 422 응답 ("운영시간 외입니다")
- 운영시간 내 장애 — 한국은행 직통 보고 + 사용자 다음 영업일 안내

### 4-3. 자동이체 결제망 자동이체 장애
- 도래 자동이체 → 다음날 재시도
- 「자동이체실행이력」(`자동이체 실행`) DELAY 상태로 적재
- 사용자 알림 발송

## 5. 복구 후 사후 처리

### 5-1. 거래 정합성 점검
- 장애 시간대 PENDING/FAILED 거래 전수 조회
- 양 은행 거래 내역 대조 (한국은행 거액결제망 의 경우 한국은행 정산 내역 대조)
- 차이 발견 시 정정 거래 발행

### 5-2. 사용자 보상
- 사용자 손해 발생 시 — 약관 §15에 의거 보상 검토
- 자동이체 미실행으로 인한 연체 발생 — 연체이자 면제 처리

### 5-3. 외부 보고
- 1시간 이상 장애 — 금감원 IT감독국 보고 (D+1 까지)
- 3시간 이상 — 금융위·한국은행 동시 보고
- 「장애 종결 보고서」 14일 이내 제출

## 6. 시스템 모니터링
- `외부 연동 상태` 테이블에 5개 외부망 헬스 60초마다 적재
- DISTINCT ON 으로 최신 상태 추출 → admin 대시보드 노출
- DEGRADED/DOWN 5분 지속 시 자동 알림 (Slack·SMS)
