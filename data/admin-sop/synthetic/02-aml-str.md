# 자금세탁방지 (AML) 및 의심거래보고 (STR) SOP

> **AUDIENCE**: ADMIN  **CATEGORY**: AML  **BASIS**: 특정금융정보법 §4·§5, KoFIU 자금세탁방지 업무규정

## 1. 의심거래 (Suspicious Transaction) 판정 기준

### 1-1. 정량 기준
- 1회 1,000만 원 이상 + 7일 이내 5회 이상 분할 거래 (Smurfing)
- 단기간 다발 입출금 — 회전율 비정상 (잔액 평균의 10배 이상)
- 거래액과 직업·연령 패턴 현저한 불일치 (예: 학생 명의 월 1억 원 이상 입금)

### 1-2. 정성 기준
- 자금 출처·사용 목적 설명 회피·일관성 없음
- 동일 IP·디바이스에서 다수 계좌 거래
- 야간·심야 시간대 거액 거래 빈도 비정상
- FATF 비협조국 송금·수취
- 가상자산 거래소 입출금 (VASP 외 거래)
- 정치적 주요인물(PEP)의 비공개 거래

## 2. 보고 절차 (STR — Suspicious Transaction Report)

### 2-1. 1단계 — 1차 검토 (영업점)
1. FDS 시스템에서 자동 룰 발동 또는 직원 직접 의심 인지
2. **거래 즉시 중단 X** — 자금 도주 우려로 관찰 우선 (특정금융정보법 §5의2)
3. 「STR 1차 검토 카드」 작성:
   - 고객 기본정보·거래 내역·의심 사유·증빙
   - 룰 발동 코드 (예: R_NIGHT, R_BURST 등)
4. 지점 컴플라이언스 담당에 전달

### 2-2. 2단계 — 본부 자금세탁방지센터 검토
1. 본부 AMLO(Anti-Money Laundering Officer) 추가 분석
2. 외부 정보 대조 — 신용정보원, 한국신용평가, 한국정보통신진흥협회
3. AI 분류기 결과 (룰+ML+LLM) 종합 점수 ≥60 → 보고 결정

### 2-3. 3단계 — KoFIU 보고
1. **거래 발생일로부터 **3영업일 이내**** (특정금융정보법 §4 ①)
2. KoFIU FIU-NET 시스템 전송 — XML 표준 양식
3. 보고 후 30일 이내 추가 거래 모니터링 강화

## 3. CTR (Currency Transaction Report — 고액현금거래보고)
- **1일 1,000만 원 이상 현금 입출금**: 익일까지 KoFIU 자동 보고 (시스템 자동)
- 분할 거래로 회피 의도 의심 시 → STR 별도 작성

## 4. 보고 면책
- 선의로 보고한 경우 형사·민사 면책 (특정금융정보법 §10)
- 고객에게 STR 보고 사실 절대 누설 금지 (Tipping-off 금지) — 위반 시 5년 이하 징역

## 5. 위반 시 제재
- STR 미보고: 최대 1억 원 과태료
- Tipping-off: 5년 이하 징역 또는 5천만 원 이하 벌금
- 임직원 개인 책임 — 형사 처벌 별도

## 6. 시스템 연동
- **AI_FDS_DECISION** 테이블 ML 분류기 결과 → STR 1차 검토 자동 트리거
- 본부 AMLO 콘솔에서 회색지대(60~80점) 케이스 사람 검토 큐 진입
- 80점 이상 자동 보고 대상은 본부 컴플라이언스 사전 승인 후 KoFIU 전송
